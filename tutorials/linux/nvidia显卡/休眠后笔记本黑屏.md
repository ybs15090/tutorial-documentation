# 休眠/睡眠唤醒后笔记本「内屏黑屏(不点亮)」修复

> 独显直连的 NVIDIA 笔记本,睡眠唤醒后内屏不通电、只有外接屏能显示。根因是 BIOS 的 deep(S3)断了内屏 eDP 链路,把睡眠模式永久改为 `mem_sleep_default=s2idle` 即可解决。

## 目标

完成本教程后,你将能够:

- [ ] 判断"睡眠唤醒内屏不亮"是否属于 deep(S3)断 eDP 链路的问题
- [ ] 用 s2idle 睡眠模式永久修复内屏唤醒不亮
- [ ] 避开 GRUB 参数粘连、DRM 状态误判等常见坑
- [ ] 理解修复后合盖的正常行为,并按需调整

## 前置条件

- 机型/环境:联想 Legion Y7000P IRH8(82YA)+ RTX 4060 + Ubuntu 20.04 / 内核 5.15 / X11 + GDM3,**独显直连(dGPU-only)** 模式(其它独显直连的 NVIDIA 笔记本通用)。
- 已安装 NVIDIA 官方驱动(本例 580 系列)。
- 具备 sudo 权限;能在内屏黑时通过**外接显示器 / SSH / `Ctrl+Alt+F3` 切 TTY** 操作。

**典型现象**:首次开机内屏 + 外屏都正常;一旦睡眠/休眠再唤醒,**内屏完全不点亮(面板不通电,不是黑屏)**,只有外接 HDMI 显示。

## 步骤

### 第一步:确认显卡架构与当前睡眠模式

```bash
lspci -nnk | grep -iEA3 "vga|3d"     # 只有 NVIDIA、无 Intel 核显(无 00:02.0)= 独显直连
cat /sys/power/mem_sleep             # 当前睡眠模式,[deep] = 走 S3(本问题根源)
```

独显直连下,内屏 eDP 由 NVIDIA 直接驱动;走 deep 时唤醒易断链路。

### 第二步:(可选,先排除)启用 NVIDIA 显存保留

NVIDIA 官方推荐的睡眠基础配置,建议保留(但**本例不是它的锅**,它针对"显存未保存花屏/崩溃",不针对"eDP 链路未重建"):

```bash
sudo tee /etc/modprobe.d/nvidia-power-management.conf >/dev/null <<'EOF'
options nvidia NVreg_PreserveVideoMemoryAllocations=1
options nvidia NVreg_TemporaryFilePath=/var/tmp
EOF
sudo systemctl enable nvidia-suspend.service nvidia-resume.service nvidia-hibernate.service
sudo update-initramfs -u && sudo reboot

# 验证(本驱动无 /sys/module/nvidia/parameters/ 目录,要看 /proc):
grep -iE "Preserve|TemporaryFile" /proc/driver/nvidia/params
```

若做完仍不亮,继续。

### 第三步:抓取故障现场,定位故障层

在"内屏黑、外屏亮"的状态下操作:

```bash
cat /sys/class/drm/card0-eDP-1/status    # connected = 面板被识别

# 取出 GUI 会话的 DISPLAY/XAUTHORITY 再看 xrandr
PID=$(pgrep -u "$(id -u)" -f 'gnome-session-binary' | head -1)
eval "export $(tr '\0' '\n' < /proc/$PID/environ | grep -E '^(DISPLAY|XAUTHORITY)=' | tr '\n' ' ')"
xrandr | grep -E "eDP|DP-|HDMI"

dmesg | grep -iE "PM: suspend entry|PM: suspend exit"   # 看本次睡眠用了哪种模式
```

- 内屏(NVIDIA 下常叫 `DP-2`)退化为 **`640x480 / 0mm` / 无 EDID** → 唤醒后 eDP 链路没重建。
- 试 `xrandr --output DP-2 --off; sleep 2; xrandr --output DP-2 --auto`:点不亮 → 驱动/链路层问题,需换睡眠模式(本例就是此情况)。

### 第四步:临时测试 s2idle

```bash
# 先重启把内屏恢复正常,再:
echo s2idle | sudo tee /sys/power/mem_sleep   # 临时切换(本次开机有效)
cat /sys/power/mem_sleep                       # 应显示 [s2idle] deep
systemctl suspend                              # 唤醒看内屏是否点亮
```

内屏正常点亮即可进入下一步固化。

### 第五步:永久固化到 GRUB

```bash
sudo cp /etc/default/grub /etc/default/grub.bak

# 整行覆盖,避免与已有参数粘连;如有其它参数(如 pci=nocrs)请一并写入、用空格隔开
sudo sed -i 's|^GRUB_CMDLINE_LINUX_DEFAULT=.*|GRUB_CMDLINE_LINUX_DEFAULT="quiet splash mem_sleep_default=s2idle"|' /etc/default/grub

grep '^GRUB_CMDLINE_LINUX_DEFAULT' /etc/default/grub   # ★确认各参数之间有空格★
sudo update-grub
sudo reboot
```

## 验证

```bash
cat /proc/cmdline           # 各参数应空格分开,含 mem_sleep_default=s2idle
cat /sys/power/mem_sleep     # 方括号应在 s2idle 上
systemctl suspend           # 反复 2~3 次(含合盖),内屏每次都点亮
```

预期输出:

```
# /proc/cmdline 片段:
... quiet splash mem_sleep_default=s2idle vt.handoff=7
# /sys/power/mem_sleep:
[s2idle] deep
```

## 注意事项

- **★GRUB 参数空格(最易踩的坑)★**:追加参数时务必与已有参数用空格隔开。曾因丢空格变成 `pci=nocrsmem_sleep_default=s2idle` 黏成一词,内核当作 `pci=` 的无效值、**直接忽略 `mem_sleep_default`**,重启后又回 deep、内屏再次不亮。改完**先 `cat /proc/cmdline` 确认参数分开**再测。
- **`/sys/class/drm/card0-eDP-1/enabled` 不可信**:NVIDIA 私有驱动下内屏亮着它也常显示 `disabled`/`dpms=Off`。判断以**肉眼 + `xrandr` 分辨率**为准(2560x1600=好 / 640x480=坏)。
- **验证 NVIDIA 参数看 `/proc/driver/nvidia/params`**,不是 `/sys/module/nvidia/parameters/`(本驱动版本无此目录)。
- **`PreserveVideoMemoryAllocations` 不是万能药**,对症(显存 vs 链路)才有效。
- **sudo 写 /etc 文件防长行折行**:`tee` heredoc 把长配置拆成多条短 `options` 行,modprobe 会自动合并。

### 合盖行为(修复后的正常现象,非故障)

| 场景 | 合盖表现 | 说明 |
|---|---|---|
| 仅内屏(无外接) | 正常休眠,开盖唤醒 | 正常 |
| 接了外屏(仅外屏/扩展/仍插着线) | **不休眠**,只关内屏继续用外屏 | 系统判定为 **docked**,`HandleLidSwitchDocked=ignore` |
| 拔掉外屏后合盖(出门) | 正常休眠 | 不再 docked |

- 小特性:出门场景开盖后屏幕有时需**敲一下键盘**才点亮(s2idle 常见,无害)。
- 想"即使接外屏合盖也休眠":`sudo sed -i 's/^#\?HandleLidSwitchDocked=.*/HandleLidSwitchDocked=suspend/' /etc/systemd/logind.conf` 后重启。

### 可选:s2idle 待机省电

s2idle 待机比 deep 费电。可开 NVIDIA S0ix 省电缓解:

```bash
echo 'options nvidia NVreg_EnableS0ixPowerManagement=1' | sudo tee -a /etc/modprobe.d/nvidia-power-management.conf
sudo update-initramfs -u && sudo reboot
```
⚠️ 会改 suspend 路径,**加完务必再测内屏**,不亮就删此行回退。

### 回滚

```bash
sudo cp /etc/default/grub.bak /etc/default/grub   # 还原 GRUB
sudo update-grub
sudo rm -f /etc/modprobe.d/nvidia-power-management.conf   # 如需移除 NVIDIA 配置
sudo update-initramfs -u && sudo reboot
```
